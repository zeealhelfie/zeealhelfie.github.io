T-Mobile Close-Out Memo Automation

Role: Internal tools & automation · Stack: Python, Google Cloud Functions, Quickbase, Box, Excel (openpyxl, pandas)

I built a cloud-based automation that takes project data from Quickbase, fills a standardized T-Mobile Close-Out Memo Excel template, and uploads the finished memo to the correct Box folder for that site. This replaced a very manual, error-prone process and turned it into a one-click workflow for project managers.

Problem

Before this automation, each close-out memo had to be created by:

  • Manually pulling data from multiple Quickbase tables
  • Copy/pasting into an Excel template with many fields
  • Checking formatting, dates, naming conventions, and file locations
  • Manually saving and uploading the file to the correct Box folder

This process was slow, inconsistent, and difficult to scale when multiple sites needed close-out memos at the same time.

Solution

End-to-End Automated Workflow

  • Trigger from Quickbase (button, Pipeline, or report)
  • Cloud Function fetches project/site data via Quickbase API
  • Python script maps FIDs to specific cells in the Excel template
  • Memo is generated and saved with a standardized filename
  • File is uploaded to the correct Box project folder
  • Quickbase can be updated with a link or status flag

Architecture

At a high level, the system looks like this:

  • Quickbase – source of truth for project, site, and client data
  • Quickbase Pipelines – triggers the automation via webhook
  • Google Cloud Functions (Gen2) – runs the Python logic
  • Excel template – formatted T-Mobile Close-Out Memo
  • Box – storage for final deliverables by project/site

Key Responsibilities

  • Designed and implemented the Python Cloud Function
  • Defined the FID → Excel cell mapping for the memo template
  • Handled date formatting, text cleanup, and validation logic
  • Integrated with Box API to place files in the correct folder
  • Collaborated with internal users to refine the template and workflow
  • Debugged deployment issues with Cloud Functions and IAM roles

Sample Code (Redacted Example)

This is a simplified example of the pattern I used; identifiers and field IDs are anonymized. 

mapping = {
    709:  "E7",
    1351: "E8",
    1372: "I8",
    244:  "I7",
}

def fill_template_from_qb(record, template_path, output_path):
    wb = openpyxl.load_workbook(template_path)
    ws = wb.active

    for fid, cell in mapping.items():
        value = record.get(str(fid))
        ws[cell] = value

    wb.save(output_path)

Impact

  • Reduced memo creation time from hours to minutes per site
  • Improved consistency across all deliverables
  • Ensured data accuracy by standardizing field mappings
  • Saved project managers significant manual effort
