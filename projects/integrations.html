Integrations & Utility Pipelines

Role: Integration engineer & internal tools · Stack: Quickbase, Pipelines, Python, APIs

Beyond the headline automations, I build smaller “glue” services and pipelines that keep data clean and systems aligned. These utilities are often what make the more visible workflows reliable in real-world use.

Editable Description Pipeline (editable-copy-of-description)

Some fields in Quickbase need to be system-controlled, but users still need a way to edit descriptions or notes. I built a pipeline to separate “system-of-record” fields from user-editable copies.

  • Creates a safe, editable copy of key description fields.
  • Keeps system-controlled values clean for use in automation and integrations.
  • Reduces the risk that users accidentally break important formulas or mappings.

Core Sync & Glue Services

Several smaller scripts keep systems in sync so that higher-level automations always have good inputs. For example:

  • Core data syncs (e.g., verizon_ariba_core_sync) to align Ariba and Quickbase.
  • Utility helpers that normalize IDs, naming conventions, and reference data.
  • Small validation tasks that run before bulk uploads or document generation.

These pieces don’t always have a UI, but they are critical to making end-to-end processes robust.

Impact

  • Reduced data integrity issues for downstream automations.
  • Made it safer for non-technical users to work in Quickbase.
  • Simplified maintenance by centralizing shared logic in small utilities.
