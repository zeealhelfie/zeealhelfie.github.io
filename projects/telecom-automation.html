Telecom Document & Compliance Automation

Role: Internal tools, document automation & compliance · Stack: Python, Quickbase, Excel/Word, Box, pandas, openpyxl

I design automations that generate carrier-required deliverables from structured project data. Instead of manually filling templates for each site, project managers can trigger a workflow that produces compliant close-out memos, SCIP workbooks, and noise studies with one click.

Key Projects

T-Mobile Close-Out Memo Automation

Cloud-based workflow that takes data from Quickbase, fills a standardized T-Mobile Close-Out Memo Excel template, and uploads the result to the appropriate Box folder.

Stack: Python · Google Cloud Functions · Quickbase API · openpyxl · Box SDK

Verizon SCIP Template Automation

Automation that pulls site and equipment data from Quickbase, cleans and transforms it with pandas, and writes into a structured Verizon SCIP workbook.

Stack: Python · Quickbase API · pandas · openpyxl · Excel templating

Telecom Noise Study Templates

Reusable noise study templates for jurisdictions like Bakersfield, Santa Rosa, and Santa Cruz that combine manufacturer noise data with local municipal code requirements.

Focus: Technical writing · dBA calculations · code/ordinance research · template design

Additional Automations

  • T-Mobile HMBP Notification & Documents (tmo-hmbp-notification) – automates the creation/notification process around HMBP-related forms.
  • T-Mobile CPUC/CLON File Automation (tmobile_cpuclon_file) – generates CPUC/CLON or similar regulatory files from project and site data.

Common Architecture Patterns

  • Quickbase as the source of truth for project, site, and equipment data.
  • Python (often on Google Cloud Functions) to transform and validate data.
  • Excel and Word templates as the base for carrier-required deliverables.
  • Box as the final storage and distribution location.

Impact

  • Reduced hours of manual document preparation per site.
  • Improved consistency and compliance across all deliverables.
  • Easier to scale when multiple sites and carriers are active at once.
  • Created a pattern for future document automations.
