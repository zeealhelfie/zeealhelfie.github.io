Verizon SCIP Template Automation

Role: Internal tools & automation · Stack: Python, Quickbase API, Excel templating, data cleanup

I built tooling around the Verizon SCIP template to automatically pull project and equipment data from Quickbase, clean and transform it, and populate a structured SCIP workbook. The goal was to reduce manual data entry, enforce consistent naming, and make it easier to submit accurate SCIP packages for Verizon sites.

Problem

The Verizon SCIP template requires a large amount of structured data: site details, equipment lists, naming conventions, and other metadata. Originally, this information was spread across multiple Quickbase tables and had to be manually copied into the template.

  • Copy/pasting from Quickbase reports into multiple SCIP tabs
  • Risk of typos, missing fields, and inconsistent naming
  • Time-consuming when processing multiple sites
  • Difficult to maintain mappings as the template evolved

Solution

Data-Driven SCIP Generator

  • Use Quickbase API to pull the necessary records into a pandas DataFrame
  • Normalize and clean the data (types, naming conventions, IDs)
  • Map Quickbase fields into specific locations in the SCIP workbook
  • Handle edge cases (missing values, mismatched types, optional fields)
  • Produce a ready-to-review SCIP file for each site

Architecture

The automation is organized around a clear separation of concerns:

  • Quickbase layer: API calls and field selection
  • Data layer: pandas DataFrames for cleaning and transformation
  • Mapping layer: dictionary that maps field labels/IDs to SCIP cells
  • Template layer: openpyxl-based writer into the Verizon SCIP workbook

Key Responsibilities

  • Defined which Quickbase fields are required for the SCIP template
  • Wrote Python functions to query Quickbase and normalize the results
  • Created mapping logic so changes in the template are easy to maintain
  • Debugged issues with field types (text vs numeric, URLs vs IDs, etc.)
  • Improved error handling to make it easier to see what’s missing or invalid

Sample Code (Redacted Example)

This example shows the pattern I use to pull Quickbase data into pandas and map it into a template. Field names and IDs are anonymized. 

import requests
import pandas as pd
import openpyxl

def qb_dataframe(headers, table_id, field_map, where):
    r = requests.post(
        "https://api.quickbase.com/v1/records/query",
        headers=headers,
        json={
            "from": table_id,
            "select": list(field_map.keys()),
            "where": where,
        },
    )
    df = pd.json_normalize(r.json()["data"])
    df.columns = df.columns.str.replace(".value", "", regex=True)

    # Rename numeric FIDs to human-readable labels
    col_names = []
    for fid in map(int, list(df.columns)):
        col_names.append(field_map[fid])
    df.columns = col_names
    return df

def fill_scip_template(row, template_path, output_path, mapping):
    wb = openpyxl.load_workbook(template_path)
    ws = wb.active

    for label, cell in mapping.items():
        ws[cell] = row.get(label)

    wb.save(output_path)

Impact

  • Reduced manual SCIP preparation time and repetitive data entry
  • Improved consistency of site naming and equipment descriptions
  • Made it easier to extend the workflow to additional projects/sites
  • Laid the groundwork for more automation around carrier deliverables
